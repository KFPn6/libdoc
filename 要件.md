# 図書館ダッシュボード 要件定義書

## 1. 概要

豊島区・新宿区・中野区の区立図書館について、複数アカウントの予約情報と返却情報を自動取得し、1画面のダッシュボードで一覧表示する。

### 対象アカウント

| 区 | アカウント数 | 表示名 |
|---|---|---|
| 豊島区立図書館 | 2 | 本人 / 家族 |
| 新宿区立図書館 | 2 | 本人 / 家族 |
| 中野区立図書館 | 1 | 本人 |

**合計 5 アカウント**

> 実名・氏名は一切表示しない。ダッシュボード上は「本人」「家族」のみとする。

---

## 2. 目的・背景

- 3区・5アカウントにまたがる予約・貸出状況を、毎回各区のマイページにログインして確認する手間を省く
- 取置済み（受け取り待ち）の見落としを防ぐ
- 返却期限の把握を容易にする
- 同じ本を複数区・複数アカウントで重複予約していないかを検知する
- スマートフォンからも1画面で確認できるようにする

---

## 3. システム構成

```
GitHub Actions（1日1回）
  ├── adapters/toshima.ts   … 豊島区 2アカウント
  ├── adapters/shinjuku.ts  … 新宿区 2アカウント
  ├── adapters/nakano.ts    … 中野区 1アカウント
  ├── merge.ts              … データ正規化・重複検知
  └── build.ts              … HTML ダッシュボード生成
        ↓
GitHub Pages（公開リポジトリ）
  └── index.html            … スマホ対応ダッシュボード
```

### 技術スタック

| 項目 | 選定 |
|---|---|
| データ取得 | Node.js + Playwright |
| 定期実行 | GitHub Actions（cron） |
| ダッシュボード配信 | GitHub Pages（公開リポジトリ） |
| 認証情報管理 | GitHub Secrets |
| 月額コスト | 0円（GitHub Free 枠内） |

### 取得頻度

- **1日1回**（GitHub Actions の cron で自動実行）
- 手動実行（workflow_dispatch）も可能とする

---

## 4. 対象図書館システム

各区とも個人の予約・貸出状況を返す公式 API は存在しない。マイページへのログインによる情報取得（ブラウザ自動操作）が必要。

| 区 | システム | ログイン先 | 利用者ID |
|---|---|---|---|
| 豊島区 | 独自サイト | https://www.library.toshima.tokyo.jp/login | 10桁（ハイフンなし） |
| 新宿区 | WINJ系（2025年1月移行） | https://opc.library.shinjuku.tokyo.jp/winj/opac/login.do | 8桁 |
| 中野区 | LiCS-Re SaaS | https://www.kn.licsre-saas.jp/tokyo-nakano/webopac/usermenu.do | 8桁 |

> 3種類の異なるシステムのため、取得ロジックは区ごとにアダプターとして分離する。

## 5. 取得データ

各区のマイページから以下の情報を取得する。

### 5.1 予約情報

| 項目 | 説明 |
|---|---|
| 書名 | 資料タイトル |
| 著者 | 著者名（取得できる場合） |
| ISBN | 資料識別用（取得できる場合） |
| 予約状態 | 順番待ち / 取置済み（受け取り待ち）など |
| 予約順位 | 順番待ちの場合の残り順位 |
| 受取館 | 取置・受取を行う図書館名 |
| 取置期限 | 取置済みの場合の受取期限 |

### 5.2 貸出（返却）情報

| 項目 | 説明 |
|---|---|
| 書名 | 資料タイトル |
| 著者 | 著者名（取得できる場合） |
| ISBN | 資料識別用（取得できる場合） |
| 返却期限 | 貸出返却の期限日 |
| 貸出館 | 貸出を行った図書館名 |

### 5.3 共通メタ情報

| 項目 | 説明 |
|---|---|
| library | 区名（toshima / shinjuku / nakano） |
| user | 表示名（本人 / 家族） |
| category | hold_ready / reservation / loan |
| fetchedAt | データ取得日時 |

---

## 6. データモデル（正規化後）

```typescript
type LibraryItem = {
  library: "toshima" | "shinjuku" | "nakano";
  user: "本人" | "家族";
  category: "hold_ready" | "reservation" | "loan";
  title: string;
  author?: string;
  isbn?: string;
  queuePosition?: number;    // 予約順位（category=reservation のとき）
  pickupLibrary?: string;    // 受取館（category=hold_ready のとき）
  pickupDeadline?: string;   // 取置期限（category=hold_ready のとき）
  returnDeadline?: string;   // 返却期限（category=loan のとき）
  loanLibrary?: string;      // 貸出館（category=loan のとき）
  status: string;            // 元サイトのステータス文字列
  isDuplicate?: boolean;
  duplicateGroup?: string;
  fetchedAt: string;
};
```

### category の定義

| 値 | 意味 | ダッシュボード上の分類 |
|---|---|---|
| hold_ready | 取置済み・受け取り待ち | 受け取り待ち |
| reservation | 順番待ち（予約中） | 予約中 |
| loan | 貸出中 | 返却予定 |

---

## 7. 重複予約検知

### 検知対象

複数のアカウント・複数の区をまたいで、**同じ資料を重複して予約している**ケースを検知する。

### 検知ロジック

1. ISBN が取得できる場合 → ISBN でグループ化
2. ISBN がない場合 → 正規化タイトル + 著者でグループ化
   - 全角/半角の統一
   - スペース・記号の除去
   - 巻数表記（「第○巻」「上・中・下」など）は別グループとして扱う
3. 同一グループに `hold_ready` または `reservation` が **2件以上** 存在する場合、`isDuplicate = true` とする

### 表示例

```
⚠️ 重複予約
「○○○」… 豊島区（本人）+ 新宿区（家族）で予約中
「△△△」… 豊島区（本人）+ 豊島区（家族）で予約中
```

## 8. ダッシュボード仕様

### 8.1 表示順序

上から以下の順にセクションを並べる。

| 順番 | セクション | 0件のときの表示 |
|---|---|---|
| 1 | 重複予約 | **非表示** |
| 2 | 受け取り待ち | 表示（「なし」と表示） |
| 3 | 返却予定 | 表示（「なし」と表示） |
| 4 | 予約中 | 表示（「なし」と表示） |

### 8.2 画面レイアウト

各セクションの見出しには、件数（N）に続けて**図書館別の内訳**を同じ行に表示する。

```
┌──────────────────────────────────────────────┐
│ 📚 図書館ダッシュボード                        │
│ 最終更新: YYYY/MM/DD HH:MM                    │
├──────────────────────────────────────────────┤
│ ⚠️ 重複予約 (N)   ← 0件なら非表示              │
│  書名 / 区 / 本人or家族                        │
├──────────────────────────────────────────────┤
│ 🔴 受け取り待ち (N) 豊島区(n)、新宿区(n)、中野区(n) │
│  書名 / 区 / 本人or家族                        │
│  受取館 / 取置期限                             │
├──────────────────────────────────────────────┤
│ 📖 返却予定 (N) 豊島区(n)、新宿区(n)、中野区(n)     │
│  書名 / 区 / 本人or家族                        │
│  返却期限（近いものは色分け）                   │
├──────────────────────────────────────────────┤
│ 🟡 予約中 (N) 豊島区(n/上限)、新宿区(n/上限)、中野区(n/上限) │
│  書名 / 区 / 本人or家族                        │
│  予約順位                                      │
└──────────────────────────────────────────────┘
```

### 8.3 図書館別の内訳表示

各セクション見出しの右に、図書館ごとの集計を「豊島区 → 新宿区 → 中野区」の順で表示する。

| セクション | 内訳の内容 | 集計対象 |
|---|---|---|
| 受け取り待ち | 各区の件数 `豊島区(n)、新宿区(n)、中野区(n)` | 本人＋家族の合算 |
| 返却予定 | 各区の件数 `豊島区(n)、新宿区(n)、中野区(n)` | 本人＋家族の合算 |
| 予約中 | 各区の `現在予約数/予約上限` `豊島区(n/20)、新宿区(n/10)、中野区(n/15)` | **本人分のみ** |

**予約中の集計ルール**

- 予約上限は 豊島区 **20冊** / 新宿区 **10冊** / 中野区 **15冊**。
- 「現在予約数」は **予約中＋受け取り待ち** の合計とする（取置済みの資料も予約枠を消費するため）。
- 上限に達している区は内訳を **赤字** で強調する。

### 8.4 表示ルール

- **利用者名**: 「本人」「家族」のみ表示（実名は表示しない）
- **区名**: 豊島区 / 新宿区 / 中野区 と表示
- **返却期限の色分け**:
  - 返却日当日 → 赤
  - 3日以内 → 黄
  - それ以外 → 通常
- **取置期限**: 期限が近いものを目立たせる
- **レスポンシブ**: スマートフォン縦画面で快適に閲覧できること（内訳が長い場合は自然に折り返す）
- **最終更新日時**: ページ上部に表示

### 8.5 アクセス方法

- GitHub Pages の URL をスマートフォンのブラウザにブックマークして利用
- 公開リポジトリで運用（GitHub Pages 無料枠を利用）

## 9. 認証情報の管理

### GitHub Secrets

リポジトリの Secrets に以下を登録する。

```
LIBRARY_ACCOUNTS  … アカウント情報（JSON）
```

### アカウント情報の形式

```json
[
  { "library": "toshima", "user": "本人", "id": "利用者ID", "pw": "パスワード" },
  { "library": "toshima", "user": "家族", "id": "利用者ID", "pw": "パスワード" },
  { "library": "shinjuku", "user": "本人", "id": "利用者ID", "pw": "パスワード" },
  { "library": "shinjuku", "user": "家族", "id": "利用者ID", "pw": "パスワード" },
  { "library": "nakano",  "user": "本人", "id": "利用者ID", "pw": "パスワード" }
]
```

> `user` フィールドの値がダッシュボード上の表示名になる。実名は使用しない。

## 11. GitHub Actions ワークフロー

```yaml
name: Fetch Library Data

on:
  schedule:
    - cron: "0 6 * * *"    # 毎日 6:00 (UTC) = 15:00 (JST)
  workflow_dispatch:        # 手動実行も可能

jobs:
  fetch:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci && npx playwright install chromium
      - run: npm run fetch-and-build
        env:
          LIBRARY_ACCOUNTS: ${{ secrets.LIBRARY_ACCOUNTS }}
      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs/
      - uses: actions/deploy-pages@v4
```

> cron の実行時刻は必要に応じて調整する。

## 13. 制約・リスク

| 項目 | 内容 | 対策 |
|---|---|---|
| 公式 API なし | スクレイピングに依存 | 区ごとアダプター分離で影響を局所化 |
| サイト構造変更 | 取得ロジックが動かなくなる可能性 | 特に新宿区は2025年1月に移行済み。変更時はアダプターのみ修正 |
| 利用規約 | 個人利用の自動取得 | 商用利用は行わない。取得頻度は1日1回に抑制 |
| 公開リポジトリ | 生成された HTML / JSON が公開される | 氏名は表示しない。書名・予約状況は含まれる点に留意 |
| GitHub Actions 無料枠 | 月 2,000 分 | 1日1回・5アカウントで十分余裕あり |
